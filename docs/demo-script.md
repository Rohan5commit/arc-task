# ArcTask demo script

## 90-second flow

### 1. Frame the product
[Open landing page]

> ArcTask is an agentic nano-payments platform for Arc. A goal becomes atomic tasks, each task gets a micro-bounty, work gets verified, and approved outputs move into a payout ledger.

### 2. Show the dashboard
[Click "Open live demo"]

Point out:
- goal composer
- task board by status
- wallet panel
- payout ledger
- activity feed

### 3. Load the seeded research workspace
[Click "Load research demo"]

Say:
> This seeded workspace already shows one task paid, one task submitted, and the rest queued. Judges can see the full lifecycle immediately.

### 4. Approve the submitted task
[In the submitted column, approve "Capture positioning claims"]

Say:
> Verification is a real gate. A task can submit evidence, but it cannot get paid until a verifier approves it.

### 5. Release a nano-payment
[In the verified column, click "Release nano-payment"]

Say:
> Once approved, ArcTask settles a nano-payment receipt into the ledger. In the hackathon build this uses a mock Arc adapter, but the receipt format mirrors what a real Arc settlement layer would emit.

### 6. Show the ledger
[Point to the ledger panel and wallet balances]

Say:
> This is the key point: agentic work is economically legible. Every finished task has a micro-bounty, a status, and a payout trail.

### 7. Create a new workflow live
[Paste a new goal into the composer and click "Decompose into tasks"]

Example goal:
> Generate a meme campaign plan for launching a new Arc-native product.

Say:
> The coordinator can also create a fresh workflow live using NVIDIA NIM. If the model is unavailable, the app falls back to deterministic planning so the demo never breaks.

## Closing line

> ArcTask shows why nano-payments matter for agentic economies: if software agents and humans are going to coordinate real work, they need a payment primitive that is small, programmable, and instant enough to track every task.
